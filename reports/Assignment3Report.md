## Phase 3: The Midterm Report (Post-Mortem)

### Plan vs. Reality

As per our initial concept summary: "Our project is an interactive visual experience centered on a diverse, infinite procedurally generated world that users can freely explore. The visual goal is to create landscapes that feel expansive and varied, including mountains, valleys, plains, and coastlines, while remaining coherent and repeatable. The world is generated through a seeded procedural generation system. Each environment is produced from a numeric seed, ensuring the same seed always recreates the same world. This makes worlds shareable without a backend and allows users to revisit discoveries by reusing the same seed as the terrain streams outward seamlessly as they travel."

Thus, our planned "Narrow Vertical Slice" would include working seeds, continuously generated terrain with no visible gaps between chunks, reliable lighting/atmospheric effects, and consistent performance near 60 FPS. 

Compared to the initial Assignment 2 MVP definition, we managed to incorporate most of the planned specs–such as fog effects, a consistent biome, seeded terrain generation, and a near 60 FPS target. As we progressed, we also adjusted other details. For example, at first we planned to use React for the UI overlay, but later, we found it more efficient to exclude it and simply stuck to using Vite for the local dev server and optimized builds. 

Additionally, rather than using libraries such as lil-gui or leva to build a user input interface which would have worked for the MVP, we opted for a custom HTML and JavaScript implementation for the UI controls that would be more flexible for further work and future features. We are proud to have also solved tricky issues like eliminating visible gaps that would appear between terrain chunks or the same seed generating inconsistent worlds.

### AI Utility

Overall, the AI tools listed in the Tech Spec saved time. Images generated by Gemini proved to act as helpful UI mockups. Additionally, coding assistance tools like VS Code’s extension for the Copilot tool in particular was very useful. It helped in deciding the file structure, writing helpful comments, and debugging errors.

### Roadmap to Final

For the final submission, we hope to eventually include the other features planned. Although the MVP currently includes a mountainous biome, we hope to offer other environment options as well. The product could also greatly benefit from weather simulation and procedural fauna or props. Currently, there is a stats overlay to monitor the FPS for debugging, but it would be nice to have an option to hide it in the final version as the end user would not really have use for it. Although difficult, a player avatar could also add to user experience and overall immersion. The camera controls also allows the user to go "beneath" the surface of the mountain surfaces and "through" the mountains themselves. Thus, ideally there would be collision restraints for a more realistic experience.

We also added javascript tests for validation the noise and random seed functions. We will likely include write more test cases for our final product. As the codebase grows, we may have to adjust our initial architecture and file structuring, but we don't anticipate it being too different in the final submission. There is also little to no lagging, which is a common issue with graphics that we will continue to be aware of and avoid.
